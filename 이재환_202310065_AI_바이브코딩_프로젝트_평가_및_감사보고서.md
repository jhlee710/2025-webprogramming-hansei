# 1. 프로젝트 평가서 (Project Evaluation Report)

## 1.1 총평 (Executive Summary)
본 프로젝트는 정적인 HTML/CSS 프로토타입(`중간고사 프로토타입`)에서 시작하여, REST API와 WebSocket을 결합한 동적 웹 애플리케이션으로 성공적으로 고도화되었습니다. 특히 `chapter2`~`chapter7`에 이르는 기초 학습 내용(시맨틱 태그, 비동기 처리, 객체 다루기)이 메인 프로젝트에 적절히 녹아들어 있습니다. AI 도구를 활용하여 생산성을 극대화하면서도, 결과물의 UI/UX 품질이 현업 수준에 근접하게 구현되었습니다.

## 1.2 상세 평가

### A. 아키텍처 및 기술 스택 (Score: A-)
*   **구조적 진화**: `중간고사 프로토타입` 폴더의 정적 구조가 메인 프로젝트(`index.html`)에서 시맨틱 마크업(`<header>`, `<main>`, `<aside>`, `<footer>`)으로 잘 개선되었습니다.
*   **데이터 흐름**: `binance.js`에서 초기 데이터는 REST API(`fetch`)로, 실시간 데이터는 WebSocket으로 처리하는 하이브리드 방식을 채택한 점은 매우 효율적인 설계입니다.
*   **알고리즘 구현**: 외부 라이브러리에 의존하지 않고 SMA(이동평균), RSI(상대강도지수)를 직접 JavaScript 함수(`calcSMA`, `calcRSI`)로 구현한 점은 AI의 도움을 받았더라도 로직 이해도 측면에서 높은 점수를 줄 수 있습니다.

### B. 코드 품질 및 스타일 (Score: B+)
*   **모던 자바스크립트**: `async/await`, `arrow function`, `const/let`, `Template Literals` 등 ES6+ 문법이 일관되게 사용되었습니다. (`chapter7/async.js` 등의 학습 흔적이 반영됨)
*   **CSS 디자인**: `style.css`에 적용된 Glassmorphism(유리 질감) 디자인과 애니메이션(`@keyframes drift`)은 시각적으로 매우 우수하며, CSS 변수(`:root`)를 활용한 유지보수성 확보도 훌륭합니다.
*   **변수 명명**: `elSymbol`, `elInterval` 등 DOM 요소 변수에 접두사를 사용하여 가독성을 높인 점이 돋보입니다.

### C. 학습 연계성 (Score: A)
*   **단계적 성장**: `Lotto numver generator`의 난수 생성 로직, `chapter7/upbitapi.js`의 API 호출 연습이 메인 프로젝트의 복잡한 로직을 소화하는 밑거름이 된 것으로 보입니다.
*   **AI 활용 능력**: 학생 본인의 보고서(`이재환...보고서.md`)에 기술된 바와 같이, 단순 복사-붙여넣기가 아니라 "요구사항 정의 -> 생성 -> 검토 -> 수정"의 프로세스를 거친 점은 시니어 엔지니어로서 긍정적으로 평가합니다.

## 1.3 개선 권고 사항
1.  **모듈화**: `binance.js`가 245줄로 비대해지고 있습니다. API 통신 로직, 지표 계산 로직, UI 렌더링 로직을 별도 파일로 분리(ES Modules 활용)하는 것을 권장합니다.
2.  **예외 처리 강화**: WebSocket 연결이 끊어졌을 때(Network Close) 자동으로 재연결(Reconnection)하는 로직이 부재합니다.

## 1.4 최종 등급
> **Grade: A (Excellent)**  
> *사유: 학부생 수준을 상회하는 UI 완성도와 실시간 데이터 처리 로직의 정확성, 그리고 AI를 도구로서 적절히 통제하며 결과물을 만들어낸 엔지니어링 역량을 높이 평가함.*

# 2. 기술 감사 보고서 (Technical Audit Report)

**분류**: 코드 감사 및 보안/안정성 점검  
**감사 대상**: 소스코드 전체 (`binance.js`, `style.css`, `index.html`)

## 2.1 코드 무결성 및 로직 검증

### 🔍 지표 계산 알고리즘 (RSI/SMA)
*   **검토**: `binance.js` 내 `calcRSI` 및 `calcSMA` 함수.
*   **발견**: RSI 계산 시 Wilder's Smoothing 기법(이전 평균을 가중치로 사용)이 적절히 구현되어 있습니다.
    ```javascript
    // binance.js line 91-92
    avgGain = (avgGain*(period-1) + g)/period;
    avgLoss = (avgLoss*(period-1) + l)/period;
    ```
*   **판정**: **적합**. 금융 보조지표 표준 수식을 정확히 따르고 있습니다.

### 🔍 메모리 관리 (Memory Leak)
*   **검토**: WebSocket 메시지 수신 시 배열 관리.
*   **발견**: `binance.js` 227번째 줄에서 데이터 배열의 길이를 제한하고 있습니다.
    ```javascript
    if(candleData.length > 500) candleData.shift();
    ```
*   **판정**: **적합**. 브라우저 메모리 폭주를 방지하는 방어 코드가 존재합니다.

## 2.2 보안 및 안정성 (Security & Stability)

### ⚠️ WebSocket 연결 안정성 (Risk: Medium)
*   **문제점**: `connectWS` 함수 내 `socket.onclose`에서 단순히 상태 메시지만 변경할 뿐, 재연결을 시도하지 않습니다. 사용자는 연결이 끊기면 페이지를 새로고침해야 합니다.
*   **권고**: `setTimeout`을 활용한 Exponential Backoff 방식의 재연결 로직 추가 필요.

### ⚠️ API 에러 핸들링 (Risk: Low)
*   **문제점**: `loadData` 함수 내 `try-catch`가 존재하지만, `fetchHistory` 실패 시 사용자에게 구체적인 원인(예: API Rate Limit 도달, 네트워크 오류 등)을 알려주지 않고 뭉뚱그려 "데이터 로드 실패"로 처리합니다.
*   **권고**: HTTP Status Code(429, 500 등)에 따른 분기 처리 필요.

### ✅ 민감 정보 노출 (Risk: None)
*   **검토**: 소스코드 내 하드코딩된 API Key 존재 여부.
*   **판정**: Binance Public Data API를 사용하므로 별도의 API Key나 Secret Key가 코드에 포함되지 않았습니다. 보안상 안전합니다.

## 2.3 성능 최적화 (Performance)

### ⚡ 렌더링 최적화
*   **현황**: `updateAnalysisAndChart` 함수가 WebSocket 메시지가 올 때마다(실시간) 호출되어 차트 전체를 다시 그리는 구조(`chart.updateSeries`)입니다.
*   **문제점**: 데이터 빈도가 높아지면 메인 스레드 부하가 발생할 수 있습니다.
*   **권고**: ApexCharts의 `updateSeries` 대신 실시간 데이터 추가에 최적화된 메서드를 사용하거나, 렌더링 주기를 `requestAnimationFrame` 등으로 스로틀링(Throttling) 할 필요가 있습니다.

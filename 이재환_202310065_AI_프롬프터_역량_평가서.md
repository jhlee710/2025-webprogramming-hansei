## 3. 프롬프트 품질 역추적 분석

결과물 코드를 기반으로 프롬프터가 AI에게 전달했을 것으로 추정되는 명령을 분석합니다.

### 🟢 잘한 프롬프트 (추정)

```
"바이낸스 API로 비트코인 캔들차트를 보여주는 웹페이지를 만들어줘.
ApexCharts를 사용하고, 디자인은 어두운 배경에 유리 질감
(Glassmorphism) 느낌으로 해줘. SMA 20일선, 60일선이랑
RSI 14일 지표도 같이 표시해줘."
```
**평가**: 기술 스택, 디자인 스타일, 기능 요구사항을 구체적으로 명시 ✓

### 🔴 부족했던 프롬프트 (추정)

```
"에러 처리도 해줘" (구체적 시나리오 없음)
또는 아예 언급 없음
```
**결과**: AI가 기본적인 `try-catch`만 적용하고 사용자 경험(UX) 고려한 처리는 생략

### 💡 개선된 프롬프트 예시

```
"바이낸스 API로 실시간 암호화폐 차트를 만들어줘.

[기능]
- ApexCharts 캔들스틱 차트 (다크 테마)
- 지표: SMA(20, 60), RSI(14)
- REST로 초기 데이터 로드, WebSocket으로 실시간 업데이트

[UI/UX]
- Glassmorphism 디자인, 반응형 레이아웃
- 로딩 중 스피너 표시

[에러 처리]  ← 이 부분이 누락됨
- 네트워크 실패 시 '재시도' 버튼 표시
- WebSocket 끊김 시 5초 후 자동 재연결
- API 요청 제한(429) 시 남은 시간 안내

[접근성]  ← 이 부분도 누락됨
- 키보드로 모든 기능 조작 가능
- 스크린리더용 ARIA 레이블 추가"
```

## 5. 최종 평가 의견

### 👍 칭찬할 점

> **"아이디어를 실제 동작하는 제품으로 만들어내는 실행력이 뛰어납니다."**

프로토타입에서 시작해 실시간 데이터 처리, 금융 지표 계산, 모던 UI까지 갖춘 완성품을 만들어낸 것은 분명한 성과입니다. AI를 단순히 "코드 생성기"가 아니라 "협업 파트너"로 활용하는 감각이 있습니다.

### 👎 냉정한 지적

> **"'동작하는 코드'와 '좋은 코드'의 차이를 이해해야 합니다."**

현재 결과물은 **"Happy Path"** (정상 작동 시나리오)에만 최적화되어 있습니다. 실제 서비스에서는 네트워크가 끊기고, 사용자가 예상치 못한 행동을 하고, 서버가 에러를 뱉습니다. 이런 상황에 대한 고려가 부족합니다.
